# 需求分析

## 1. 引言

### 1.1 背景

千时百岗社区活动管理系统是为了更好地管理参与者参加活动的情况信息而设计的。现今，社区还未使用软件来统一管理社区党政活动。传统方式通过文本、表格管理社区活动，人工处理数据信息工作量大，容易出错，数据繁多不易查找，缺乏系统规范的信息管理方式。

基于这些问题，我们认为有必要建立一个社区管理系统，使社区的管理工作更加系统规范，信息的查询速度和准确性更快更高，方便社区工作人员开展活动、管理活动和参与者；同时也方便参与者查询，参与活动。能够有效提升社区的管理水平。



### 1.2 目的

千时百岗社区活动管理系统是一个界面简洁、操作简单、功能齐全的系统。前台使用微信小程序，方便社区管理人员及时发布活动，用户随时查看社区活动的信息等。后台使用网页端和小程序端的结合，一些基本的活动管理可以在小程序端完成，网页端提供更加全面，完善的管理功能。这样设计既是为了更加高效、规范地实现对社区的管理，又是为了方便用户及时查看社区信息，保证信息的时效性和高效性。

千时百岗小程序提供了租户功能，用户可以自主的申请租用我们的系统，得到租用批准后，便可以在系统中对社区的活动进行发布、管理等。

此社区活动管理系统应达到的目标：

1. 界面设计简洁大方，操作简单，效率高，安全性高，同时便于维护和管理。
2. 社区人员通过前台登录，可以浏览相关活动、参与活动、查看公告等。
3. 社区管理人员可以租用系统，通过后台登录后，可以发布活动、管理社区人员等。
4. 系统管理员可以对社区进行统一管理，租用批准等。

## 3. 非功能需求

### 3.1 性能需求

响应时间：能让用户拥有流畅的使用体验，不可出现页面间跳转时间过长，使用过程中频繁出现卡顿等现象。可以使用数据分页加载，每次只加载部分数据，当用户进行进一步操作时，再逐步加载新数据。当某个页面数据量较大，又无法进行优化时，可以给用户提供加载进度条、预加载时间等。

吞吐量：在活动高峰期，有大量社区人员使用系统时，要保证系统可以正常的运行，能够承受一定数量级的请求量。



### 3.2 安全性

访问权限控制：设置用户的访问权限，不可出现权限与人员不相符的情况。超级管理员，社区管理员、社区人员各自拥有不同的权限。

系统安全：防止系统信息和用户信息的泄露，对数据进行加密保护，特别是对隐私数据加强保护。应定期对数据进行备份，可以采用本地、异地、跨地等数据保护策略。



### 3.3 可扩展性

系统能够根据业务需求，在不影响其他业务功能的前提下，方便扩展新的功能。采用模块化设计，流程支持灵活配置，减少改动工作量。降低各功能间的耦合度，使功能模块之间互不影响。并且功能具有可复用性，通用内容可复用，避免编写重复的功能代码。以及系统的运行环境也要几倍可扩展性，能够在不同的操作系统间进行使用。
